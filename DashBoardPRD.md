Executive Summary### 1.1 PurposeThis document defines the complete product requirements for "Yohan's Academic Dashboard," a personal family web application designed to comprehensively track and manage the academic journey of a 10th-grade student following the IGCSE curriculum in India for the academic year 2026-27.### 1.2 ScopeThe dashboard will serve as a centralized platform for tracking:- Academic subjects, topics, and resources- Assignments and deadlines- Test performance and analytics- Attendance records- School fee payments- Events and participation- Awards and achievements- Extracurricular activities- Personal notes and communications### 1.3 Target AudienceThis is a private, family-only application intended for 3-4 users:- Parents (primary administrators)- Student (Yohan)- Extended family member (optional, view-only)### 1.4 Key Value Propositions| Value | Description ||-------|-------------|| **Centralized Tracking** | Single source of truth for all academic information || **Performance Insights** | Visual analytics to identify trends and areas for improvement || **Portfolio Building** | Documented achievements ready for future applications || **Deadline Management** | Proactive reminders for assignments, tests, and payments || **Historical Record** | Comprehensive archive of the entire academic year |---## 2. Project Overview### 2.1 Project Information| Attribute | Detail ||-----------|--------|| **Project Name** | Yohan's Academic Dashboard 2026-27 || **Project Type** | Personal/Family Web Application || **Academic Context** | IGCSE 10th Grade || **School Location** | India || **Academic Year** | 2026-27 (June 2026 - April 2027) || **Exam Session** | May/June 2027 (Cambridge International) |### 2.2 BackgroundThe IGCSE (International General Certificate of Secondary Education) 10th grade is a critical academic year that:- Culminates in board examinations- Requires systematic preparation across multiple subjects- Generates documentation important for future academic applications- Involves significant coordination between student, parents, and schoolA dedicated tracking system addresses the complexity of managing this information effectively.### 2.3 Problem StatementCurrently, academic information is scattered across:- School communication portals- Physical documents and receipts- Email communications- Personal memory- Various notes and papersThis fragmentation leads to:- Missed deadlines and forgotten assignments- Difficulty tracking performance trends- Incomplete records for future reference- Reactive rather than proactive academic management### 2.4 Proposed SolutionA unified web-based dashboard that:- Consolidates all academic information in one place- Provides visual insights into performance trends- Sends proactive reminders for important deadlines- Creates a comprehensive record for future use- Enables family collaboration in supporting the student---## 3. Goals and Objectives### 3.1 Primary Goals| Goal ID | Goal | Success Criteria ||---------|------|------------------|| G1 | Comprehensive Academic Tracking | 100% of subjects, assignments, and tests logged || G2 | Performance Visibility | Clear visualizations showing trends across terms || G3 | Deadline Management | Zero missed deadlines due to lack of awareness || G4 | Portfolio Documentation | All achievements documented with evidence || G5 | Financial Tracking | Complete record of all fee payments |### 3.2 Secondary Goals| Goal ID | Goal | Success Criteria ||---------|------|------------------|| G6 | Family Collaboration | All family members actively using the platform || G7 | Self-Reflection Support | Regular notes and reflections logged || G8 | Exam Preparation Tracking | Systematic revision progress monitoring || G9 | Attendance Monitoring | Early identification of attendance issues || G10 | Communication Records | All PTM notes and important communications logged |### 3.3 Objectives**Quantitative Objectives:**- Reduce time spent searching for academic information by 80%- Achieve 100% logging compliance for all academic activities- Generate term-end reports within 5 minutes- Maintain data for at least 5 years post-academic year**Qualitative Objectives:**- Improve family visibility into academic progress- Support data-driven conversations about performance- Build a valuable portfolio for future applications- Create a positive, organized academic management experience---## 4. =============================================-- DATABASE SCHEMA FOR YOHAN'S ACADEMIC DASHBOARD-- =============================================-- Enable UUID extensionCREATE EXTENSION IF NOT EXISTS "uuid-ossp";-- =============================================-- USERS TABLE-- =============================================CREATE TABLE users (    user_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),    email VARCHAR(255) UNIQUE NOT NULL,    password_hash VARCHAR(255) NOT NULL,    full_name VARCHAR(100) NOT NULL,    role VARCHAR(20) NOT NULL CHECK (role IN ('admin', 'student', 'viewer')),    avatar_url TEXT,    is_active BOOLEAN DEFAULT true,    last_login TIMESTAMP,    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);-- =============================================-- SUBJECTS TABLE-- =============================================CREATE TABLE subjects (    subject_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),    subject_name VARCHAR(100) NOT NULL,    subject_code VARCHAR(20),    subject_type VARCHAR(20) NOT NULL CHECK (subject_type IN ('Core', 'Elective')),    exam_board VARCHAR(20) CHECK (exam_board IN ('Cambridge', 'Edexcel', 'Other')),    target_grade VARCHAR(5) CHECK (target_grade IN ('A*', 'A', 'B', 'C', 'D', 'E', 'F', 'G')),    predicted_grade VARCHAR(5) CHECK (predicted_grade IN ('A*', 'A', 'B', 'C', 'D', 'E', 'F', 'G')),    color_code VARCHAR(7),    description TEXT,    is_active BOOLEAN DEFAULT true,    created_by UUID REFERENCES users(user_id),    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);-- =============================================-- FACILITATORS TABLE-- =============================================CREATE TABLE facilitators (    facilitator_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),    facilitator_name VARCHAR(100) NOT NULL,    email VARCHAR(255),    phone VARCHAR(20),    office_hours VARCHAR(200),    notes TEXT,    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);-- Junction table for facilitators and subjectsCREATE TABLE facilitator_subjects (    facilitator_id UUID REFERENCES facilitators(facilitator_id) ON DELETE CASCADE,    subject_id UUID REFERENCES subjects(subject_id) ON DELETE CASCADE,    PRIMARY KEY (facilitator_id, subject_id));-- =============================================-- BOOKS TABLE-- =============================================CREATE TABLE books (    book_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),    subject_id UUID REFERENCES subjects(subject_id) ON DELETE CASCADE,    book_title VARCHAR(200) NOT NULL,    author VARCHAR(100),    publisher VARCHAR(100),    isbn VARCHAR(20),    edition VARCHAR(50),    publication_year INTEGER,    book_type VARCHAR(20) CHECK (book_type IN ('Textbook', 'Workbook', 'Guide', 'Reference')),    purchase_status VARCHAR(20) NOT NULL CHECK (purchase_status IN ('Owned', 'Borrowed', 'Digital', 'To_Purchase')),    cost DECIMAL(10, 2),    purchase_date DATE,    notes TEXT,    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);-- =============================================-- TOPICS TABLE-- =============================================CREATE TABLE topics (    topic_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),    subject_id UUID REFERENCES subjects(subject_id) ON DELETE CASCADE,    book_id UUID REFERENCES books(book_id) ON DELETE SET NULL,    chapter_number INTEGER,    chapter_name VARCHAR(100),    topic_name VARCHAR(200) NOT NULL,    topic_order INTEGER,    weightage_percentage DECIMAL(5, 2),    difficulty_level VARCHAR(10) CHECK (difficulty_level IN ('Easy', 'Medium', 'Hard')),    completion_status VARCHAR(20) NOT NULL DEFAULT 'Not_Started'         CHECK (completion_status IN ('Not_Started', 'In_Progress', 'Completed', 'Needs_Revision')),    date_covered DATE,    class_notes_url TEXT,    resource_urls TEXT[], -- Array of URLs    personal_notes TEXT,    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);-- s.subject_id,    s.subject_name,    s.target_grade,    s.predicted_grade,    COUNT(DISTINCT t.topic_id) as total_topics,    COUNT(DISTINCT CASE WHEN t.completion_status = 'Completed' THEN
